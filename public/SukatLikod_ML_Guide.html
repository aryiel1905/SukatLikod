Section 01 — Core Concept

How the Model Actually Works

The model does not process raw photos directly. Instead, photos are used to extract numeric posture features — measurements describing the person's position. Those numbers become the training data.

Key Idea: A photo → keypoints → numeric features → label. The model learns from the numbers, not the pixels.
✓ Proper
vs
⚠ Needs Correction

The 4 numeric features extracted from each frame:

Feature What it measures Unit
trunk_angle How far the torso leans forward from vertical degrees
head_forward How far the head protrudes ahead of the spine baseline pixels / ratio
shoulder_tilt Height difference between left and right shoulder keypoints degrees
trunk_variance Variability of trunk position over recent frames float

Section 03 — Folder Setup

Project Folder Structure

Create these folders in the project root before you start. The structure keeps raw material, labeled data, and model files clearly separated.

├── dataset/
│ ├── raw_videos/ ← your original video clips go here

Section 04 — Labeling Rules

Labeling Your Frames

Define your labeling rules before you start and keep them consistent throughout. Moving a frame into the right folder is all you need to do — no special tools required.

✅
proper/
  • Neutral, upright posture
  • Head not significantly forward
  • Shoulders roughly level
  • Full body visible
⚠️
needs_correction/
  • Visible slouch or hunch
  • Clear forward head posture
  • Noticeably uneven shoulders
  • Body clearly visible
🚫
skip/
  • Blurry or out of focus
  • Body partially occluded
  • Wrong camera angle
  • Any uncertainty
Tip: When in doubt, put the frame in skip/. Clean data matters more than dataset size. An uncertain label is worse than no label.
Section 05 — Data Collection

Collecting and Extracting Video Data

Aim for variety — different people, lighting conditions, and camera distances all help the model generalize better.

Recommended starting target: 20–40 clips, roughly balanced between proper and needs_correction classes.

Once you have raw videos in dataset/raw_videos/, extract frames using ffmpeg at 2 frames per second. This keeps adjacent frames different enough to avoid near-duplicate training samples. 

# Extract frames from a clip at 2 fps
# Replace clip01 with your actual filename
ffmpeg -i dataset/raw_videos/clip01.mp4 -vf fps=2 dataset/frames/clip01_%04d.jpg

# Repeat for each video clip
ffmpeg -i dataset/raw_videos/clip02.mp4 -vf fps=2 dataset/frames/clip02_%04d.jpg

After extraction, open the dataset/frames/ folder and manually move each file into proper/, needs_correction/, or skip/ according to your labeling rules above.

Section 06 — Dataset CSV

Building posture_dataset.csv

Create the file at backend/data/posture_dataset.csv. Each row represents one labeled frame with its 4 computed feature values.

trunk_angle head_forward shoulder_tilt trunk_variance label
3.2 0.05 1.1 0.008 proper
18.7 0.31 6.4 0.045 needs_correction
22.1 0.27 8.0 0.062 needs_correction
4.5 0.07 0.9 0.010 proper

You can fill values manually if you have measurements, or use a feature-extraction script to compute them from your labeled frames automatically. See backend/data/posture_dataset.sample.csv for a complete reference.

Section 07 — Training the Model

First-Time Setup & Training

Run these commands once to set up your Python environment and train the initial model. 

# 1. Create a virtual environment
python -m venv .venv

# 2. Activate it (Windows)
.venv\Scripts\activate

# 3. Install dependencies
pip install -r backend/requirements.txt

# 4. Train the model
python backend/train.py --data backend/data/posture_dataset.csv

After training, two output files will be created in backend/model/:

🧠
posture_model.joblib
The serialized trained model used for live predictions
📋
model_meta.json
Metadata: feature names, class labels, training timestamp

Recommended retrain command (works even without activating venv):

.\.venv\Scripts\python.exe backend/train.py --data backend/data/posture_dataset.csv
Section 08 — Running the API & Frontend

Connecting the Model to the App

Start the inference API, then connect the React frontend to it via environment variable. 

# Start the FastAPI inference server
uvicorn backend.api:app --reload --host 127.0.0.1 --port 8000

# Health check endpoint (open in browser)
http://127.0.0.1:8000/health

Create a .env file in the project root:

# .env
VITE_ML_API_URL=http://127.0.0.1:8000

Then run the frontend:

npm run dev

The app will automatically call the /predict endpoint with extracted posture features and display the result.

Production note: In Vercel, set VITE_ML_API_URL to your Render backend URL (example: https://sukatlikod.onrender.com).
Section 09 — Retraining Workflow

The 7-Step Improvement Cycle

Retraining is expected and should happen repeatedly as your dataset grows and improves. Follow this cycle to continuously improve model accuracy.

🎥
Gather Videos
More clips, new people, varied conditions
🖼️
Extract Frames
ffmpeg at 2 fps per clip
🏷️
Label Frames
Move into proper / needs / skip
📊
Update CSV
Add new rows to posture_dataset.csv
🧠
Retrain
python backend/train.py
🚀
Run & Test
Start API, open app, check predictions
🔍
Review Errors
Fix wrong labels, improve data

For most retraining sessions, the only command you need is:

# Activate venv (if not already active), then retrain
.venv\Scripts\activate
python backend/train.py --data backend/data/posture_dataset.csv

Section 11 — Automation Tips

Using Codex to Speed Things Up

Codex can automate the repetitive parts of this pipeline. Ask it to do one clear task at a time, then verify the output before moving on.

Useful tasks to delegate:

Generate feature-extraction script from labeled frames Create a labeling helper utility Validate class balance in posture_dataset.csv Summarize training metrics after each run Write a batch frame-extraction shell script Auto-split dataset into train/validation sets
Best practice: Give Codex one well-defined task per prompt. Verify each output before moving to the next step — chaining unverified scripts can corrupt your dataset.
